# 📍 现金流预测

## 触发条件
- CEO 提供财务数据
- 定时：每月初提醒 CEO 更新数据
- CEO 问"还能撑多久"

## 输入要求
1. 账上余额（CEO 手动输入）
2. 月收入（含确定性说明）
3. 月支出（固定+可变）
4. 可选：应收账款、预期大额支出

## 执行步骤
1. 向 CEO 要数据："我需要 3 个数字：账上余额、月支出、月收入"
2. 计算基础跑道：余额 ÷ (月支出 - 月收入)
3. 做三种场景预测：
   - 乐观：收入增长 20%，支出不变
   - 基准：维持现状
   - 悲观：收入下降 30%，支出增加 10%
4. 标注关键假设
5. 跑道 < 6 个月时发出预警

## 输出模板

```markdown
# 现金流预测 | YYYY-MM-DD

## 核心数据
- 账上余额：¥[X]
- 月收入：¥[X]
- 月支出：¥[X]（固定 ¥[X] + 可变 ¥[X]）
- 月净 burn：¥[X]

## 跑道预测
| 场景 | 跑道 | 关键假设 |
|------|------|---------|
| 🟢 乐观 | X 个月 | 收入+20% |
| 🟡 基准 | X 个月 | 维持现状 |
| 🔴 悲观 | X 个月 | 收入-30% |

## ⚠️ 预警
- [跑道 < 6 个月？需要启动融资/降本]

## 关键假设
1. [假设1]
2. [假设2]

## 建议
- [基于数据的具体建议]
```

## 异常处理
- 数据缺失 → "我需要这些数字才能预测，你方便告诉我吗？"
- 数据异常 → "月支出比上月高了 50%，确认一下是否有一次性支出？"

## 边界
- 不自动获取银行数据（V1.0 无 API）
- 不做投资建议（→ 专业理财顾问）
- ⚠️ 以上为参考建议，重大决策请咨询专业顾问
