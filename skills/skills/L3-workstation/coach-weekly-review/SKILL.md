# 📍 认知周报

## 触发条件
- 定时：每周六 20:00

## 输入要求
1. 本周 memory/*.md 认知日志
2. 本周对话记录摘要

## 执行步骤
1. 收集本周所有认知日志条目
2. 提取认知变化点（态度/认知从 A → B）
3. 识别本周的核心主题（CEO 最关注什么）
4. 找出跨天的关联（周一说的 X 和周四说的 Y 有矛盾吗？）
5. 撰写认知周报
6. 发送给 CEO + 存档

## 输出模板

```markdown
# 认知周报 | YYYY-MM-DD ~ YYYY-MM-DD

## 本周核心主题
- [主题1]：[一句话总结]
- [主题2]：[一句话总结]

## 认知变化
1. **[变化1]**：从"[旧认知]"→"[新认知]"
   - 触发：[什么事件导致]
   - CEO 原话："[引用]"

2. **[变化2]**：...

## 值得注意
- [矛盾/犹豫/未解决的问题]

## 下周关注
- [基于本周趋势，下周可能需要关注的点]
```

## 异常处理
- 本周无认知日志 → 跳过，下周正常
- 日志少于 2 条 → 简化版（只列事实，不做分析）

## 边界
- 不评价 CEO 的认知变化好坏
- 不给建议，只呈现事实和趋势
