# 📍 SOP 文档化

## 触发条件
- CEO 口述了一个流程
- 同一流程被问/解释 2 次以上
- 新人入职需要操作手册
- CEO 说"这个事情每次都要重复说"

## 输入要求
1. CEO 口述的流程步骤
2. 适用场景和触发条件
3. 可选：异常情况处理、历史对话中的相关信息

## 执行步骤
1. **提取流程**：从 CEO 口述/历史对话中整理步骤
2. **结构化**：按 SOP 模板格式化
3. **补全缺失**：列出缺失环节请 CEO 补充
4. **消歧**：有歧义的步骤确认
5. **CEO 确认**：确认后归档
6. **定期更新**：流程变化时更新版本

## 输出模板
```
## SOP：{流程名称}

### 基本信息
- 适用场景：{什么时候用}
- 触发条件：{什么触发这个流程}
- 责任人：{谁执行}
- 预计耗时：{时间}

### 前置条件
- [ ] {条件1}
- [ ] {条件2}

### 操作步骤
| 步骤 | 操作 | 注意事项 | 产出 |
|------|------|---------|------|
| 1 | | | |
| 2 | | | |
| 3 | | | |

### 异常处理
| 异常情况 | 处理方式 |
|----------|---------|
| | |

### 完成标准
- [ ] {标准1}
- [ ] {标准2}

### 版本记录
| 版本 | 日期 | 变更 |
|------|------|------|
| 1.0 | | 初版 |
```

## 异常处理
- 流程不完整 → 列出缺失环节请 CEO 补充
- 流程有歧义 → "这两步的先后顺序是？"
- CEO 懒得确认 → 先标注"待确认"发出，下次用到时再核实

## 边界
- 不做技术操作手册（→ CTO）
- 不做产品使用文档（→ CPO）
- 不做人事制度（→ CHRO）
