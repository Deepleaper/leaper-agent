# 📍 偏好学习

## 触发条件
- 对话中识别到偏好信号（时间、长度、风格）
- CEO 明确表达偏好（"以后别这样..."）
- 行为模式连续 3 次以上一致

## 输入要求
1. CEO 行为模式（回复时间、消息长度、阅读习惯）
2. CEO 明确表达的偏好
3. 历史偏好记录

## 执行步骤
1. 观察并记录 CEO 行为信号：
   - 回复速度（哪些话题秒回？哪些忽略？）
   - 消息长度偏好（喜欢长分析还是短结论？）
   - 时间偏好（几点活跃？几点不想被打扰？）
   - 内容偏好（喜欢数据还是故事？表格还是文字？）
2. 单次信号只记录观察，不更新偏好
3. 同一信号出现 3 次以上 → 更新 CEO 偏好
4. 同步全局画像，其他 CXO 自动适配

## 输出模板

```markdown
## 偏好更新 | YYYY-MM-DD

**偏好类型：** [时间/格式/内容/频率]
**观察证据：**
- [日期1]：[行为]
- [日期2]：[行为]
- [日期3]：[行为]
**结论：** [CEO 偏好 X 而非 Y]
**影响范围：** [哪些 CXO 需要适配]
**同步状态：** [已/未 同步全局画像]
```

## 异常处理
- 偏好变化不确定 → 先记录观察，不更新
- 偏好与之前矛盾 → 可能是情境性的，标注"待确认"
- CEO 明确表达的偏好 → 立即更新，不需要 3 次验证

## 边界
- 不追问 CEO 的偏好（通过观察获得）
- 不暴露"我在学习你的偏好"
- 偏好数据不泄露给外部
