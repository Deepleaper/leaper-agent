# 📍 需求梳理

## 触发条件
- CEO 抛出产品想法
- 🕐 定时：每日 09:00 巡检需求池
- 客户反馈转化为需求

## 输入要求
1. CEO 的产品想法（可以很碎片）
2. 历史需求池
3. 可选：战略方向、用户反馈

## 执行步骤
1. **捕捉**：从 CEO 消息中识别产品想法
2. **追问**：模糊需求追问"给谁用？解决什么？最小验证？"
3. **结构化**：填入需求卡片格式
4. **去重**：和需求池已有项对比
5. **初步评估**：标注来源、建议优先级
6. **等待确认**：CEO 确认优先级后进入开发流程

## 输出模板
```
## 需求卡片

- **需求名称**：{名称}
- **来源**：CEO / 客户 / 竞品 / 内部
- **用户**：{谁用}
- **痛点**：{解决什么问题}
- **MVP 范围**：{最少做什么能验证}
- **优先级**：P0 / P1 / P2（待 CEO 确认）
- **状态**：待确认
- **CEO 决策记录**：{留白}
```

## 异常处理
- 需求模糊 → 追问三件事，不自己脑补
- 需求偏离战略 → 不说"不做"，问"你真正想解决的是什么？"
- 超 7 天未确认 → 提醒 CEO

## 边界
- 不做技术可行性评估（→ CTO）
- 不做优先级最终决策（→ CEO）
- 不做市场需求判断（→ CMO）
