# 📍 竞品产品分析

## 触发条件
- 🕐 定时：每周三 10:00 扫描
- CEO 提到竞品产品动态
- 需求决策时需要竞品参考

## 输入要求
1. 竞品名称和关注点
2. 我方产品现状
3. 可选：竞品历史功能清单

## 执行步骤
1. **信息收集**：搜索竞品最新产品更新
2. **功能拆解**：对手做了什么、怎么做的
3. **对比分析**：和我方产品的差异
4. **影响判断**：对我方的威胁和机会
5. **建议**：跟/不跟/差异化，给理由

## 输出模板
```
## 竞品产品分析：{竞品名}

### 最新动态
{竞品做了什么}

### 功能对比
| 功能 | 竞品 | 我方 | 差距 |
|------|------|------|------|
| | ✅/❌ | ✅/❌ | |

### 产品策略判断
{竞品的产品方向是什么}

### 对我方影响
- 威胁：{什么}
- 机会：{什么}

### 建议
{跟/不跟/差异化} — 理由：{理由}
```

## 异常处理
- CEO 因竞品焦虑 → 用事实降温，不说"别担心"
- 竞品信息不足 → 标注信息来源和置信度
- 和 CMO/CTO 竞品分析结论冲突 → 列出各视角差异

## 边界
- 不做技术栈分析（→ CTO）
- 不做市场份额/融资分析（→ CMO）
- 不做竞品财务分析（→ CFO）
