# 📍 PRD 撰写

## 触发条件
- 需求状态变为"已确认"
- CEO 说"这个可以做了，出个 PRD"
- CTO 需要产品文档开始评估

## 输入要求
1. 确认后的需求卡片
2. 用户故事（来自 user-stories 工位）
3. 产品定位和战略方向
4. 可选：历史 PRD 风格、技术约束

## 执行步骤
1. **回顾需求**：确认需求卡片和 CEO 决策记录
2. **用户故事展开**：核心用户故事 + 边缘场景
3. **功能设计**：每条可验收的功能点
4. **边界划定**：明确"不做"列表
5. **验收标准**：写成 checklist
6. **数据埋点**：定义要追踪的指标
7. **CEO 审核**：确认后同步 CTO

## 输出模板
```
## PRD：{功能名称}

### 版本信息
- 版本：v{N}
- 日期：{日期}
- 状态：草稿/已确认
- 优先级：P{N}

### 一句话
{这个功能是什么，用一句话说清}

### 用户故事
- 作为{谁}，我想{做什么}，这样{得到什么}

### 核心功能
1. {功能点1}（验收：{标准}）
2. {功能点2}（验收：{标准}）
3. {功能点3}（验收：{标准}）

### 不做（本版本）
- {明确排除的功能}

### 交互流程
{用户操作路径描述}

### 验收标准
- [ ] {标准1}
- [ ] {标准2}
- [ ] {标准3}

### 数据埋点
| 事件 | 触发条件 | 追踪目的 |
|------|---------|---------|
| | | |

### 变更记录
| 日期 | 变更内容 | 原因 |
|------|---------|------|
| | | |
```

## 异常处理
- CEO 审 PRD 时改需求 → 不硬顶不全改，给折中方案让 CEO 选
- CTO 说技术不可行 → 调整方案，记录技术约束
- 所有变更写入变更记录

## 边界
- 不做技术方案设计（→ CTO）
- 不做 UI 视觉设计（→ 设计师）
- 不做市场定位决策（→ CMO）
