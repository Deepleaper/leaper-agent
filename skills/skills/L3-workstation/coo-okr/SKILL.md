# 📍 OKR 追踪

## 触发条件
- 🕐 定时：每周一 09:00 检查进度
- 季度初建立/调整 OKR
- CEO 问"我们目标完成得怎么样"

## 输入要求
1. CEO 口述的目标和关键结果
2. 当前进度（CEO 手动更新）
3. 可选：历史 OKR 数据

## 执行步骤
1. **建立框架**：帮 CEO 把模糊目标变成 O+KR 格式
2. **合理性检查**：KR 是否可量化、有 deadline、3-5 个为宜
3. **每周检查**：周一提醒 CEO 更新进度
4. **偏差预警**：进度落后时主动提醒
5. **季度复盘**：季末评估完成度，为下季度输入

## 输出模板
```
## OKR 追踪 Q{N}

### O1：{目标}
| KR | 目标值 | 当前值 | 进度 | 状态 |
|----|--------|--------|------|------|
| KR1 | | | {%} | 🟢/🟡/🔴 |
| KR2 | | | {%} | |

### O2：{目标}
| KR | 目标值 | 当前值 | 进度 | 状态 |
|----|--------|--------|------|------|

### 偏差预警
- 🔴 {KR名}：落后 {N}%，建议{动作}

### 本周提醒
- {需要 CEO 更新的项}
```

## 异常处理
- CEO 不更新 OKR → 温和提醒"上周 OKR 有更新吗？一句话就行"
- 目标过多 → 建议收窄到 3 个 O，每个 3-5 个 KR
- KR 不可量化 → 帮转换为可量化指标

## 边界
- 不做战略方向判断（→ CEO Coach / CSO）
- 不做具体业务执行（→ 对应 CXO）
