# 📍 用户声音

## 触发条件
- 🕐 定时：每周一汇总
- CEO 转来用户反馈
- 公开渠道出现用户评论

## 输入要求
1. 用户反馈内容（CEO 转来/公开渠道）
2. 反馈来源和用户背景
3. 可选：历史反馈数据

## 执行步骤
1. **收集**：汇总本周所有用户反馈
2. **分类**：Bug / 功能需求 / 体验问题 / 赞美
3. **聚类**：相似反馈合并，统计出现次数
4. **Top N 提取**：按频次×严重度排序
5. **建议**：每条 Top 反馈给处理建议
6. **转需求**：值得做的转入需求池

## 输出模板
```
## 用户声音周报 {日期}

### 本周反馈汇总
- 总量：{N} 条
- Bug：{N} | 需求：{N} | 体验：{N} | 赞美：{N}

### Top 3 反馈
| # | 内容 | 出现次数 | 类型 | 建议 |
|---|------|---------|------|------|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |

### 转入需求池
- {需求名}（来源：用户反馈 ×{N}）

### 本周用户情绪
{正面/中性/负面} — {一句话总结}
```

## 异常处理
- CEO 转来个别反馈 → 不急着改，先量化"个例还是普遍"
- 负面情绪集中爆发 → 紧急上报 CEO，区分产品问题还是技术问题
- 反馈与战略矛盾 → 标注矛盾点让 CEO 决策

## 边界
- 不做技术 Bug 修复（→ CTO）
- 不做客户关系维护（→ CRO/CMO）
- 不做公关危机处理（→ CCO）
