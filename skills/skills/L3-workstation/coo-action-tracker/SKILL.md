# 📍 行动项追踪

## 触发条件
- 每次对话后自动提取 action items
- 🕐 定时：每日检查逾期项
- CEO 问"我还有什么事没做"

## 输入要求
1. 对话内容（自动提取）
2. 历史 action items 列表
3. 可选：优先级标注

## 执行步骤
1. **提取**：从对话中识别 TODO/承诺/决策
2. **结构化**：内容+责任人+Deadline+状态
3. **去重**：和历史列表合并，避免重复
4. **追踪**：按盯人协议定期检查
5. **闭环**：完成的标记 ☑，取消的标记原因

## 输出模板
```
## Action Items 看板

### ⏰ 逾期（需立即处理）
| # | 内容 | 责任人 | Deadline | 逾期天数 |
|---|------|--------|----------|---------|
| | | | | |

### 🔄 进行中
| # | 内容 | 责任人 | Deadline | 状态 |
|---|------|--------|----------|------|
| | | | | |

### ☐ 待办
| # | 内容 | 责任人 | Deadline |
|---|------|--------|----------|
| | | | |

### ☑ 本周完成
| # | 内容 | 完成日期 |
|---|------|---------|
| | | |
```

## 异常处理
- 逾期 1 天 → 直接问进展
- 逾期 3 天 → "要不要调整 deadline？"
- 逾期 7 天 → "要不要重新评估这件事还做不做？"
- CEO 连续不回应 → 降频但保留记录，不删除

## 边界
- 不做优先级决策（→ CEO 决策）
- 不替代执行（→ 对应 CXO）
- 只追踪"事"的进度（人的成长 → CEO Coach）
