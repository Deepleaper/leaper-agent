# 📍 定位和 Messaging

## 触发条件
- CEO 问"怎么介绍自己""我们到底是做什么的"
- 产品重大更新、进入新市场、融资路演准备
- Messaging 不一致（团队对外说法不统一）

## 输入要求
1. 产品核心功能和差异化点
2. 目标客户群（行业、规模、角色）
3. 竞品定位（至少 2-3 家）
4. 可选：现有 slogan/one-liner、客户反馈

## 执行步骤
1. **现状诊断**：收集现有对外材料，找出不一致的地方
2. **客户倒推**：从理想客户的痛点出发，而非产品功能
3. **竞品卡位**：画定位地图，找到空白区间
4. **五层定位**：依次确定↓
5. **Messaging 矩阵**：不同场景不同说法
6. **CEO 确认**：让 CEO 用这个 one-liner 介绍一次，不顺口就改

## 输出模板
```
## 定位文档 v{版本}

### 1. One-liner（一句话）
{公司名}是{什么}，帮{谁}解决{什么问题}。

### 2. 我们是什么
{2-3 句话展开}

### 3. 我们不是什么
- 不是{X}
- 不是{Y}

### 4. ICP（理想客户画像）
| 维度 | 描述 |
|------|------|
| 行业 | |
| 规模 | |
| 决策人 | |
| 痛点 | |
| 预算 | |

### 5. 为什么选我们（差异化壁垒）
1. {壁垒1}
2. {壁垒2}
3. {壁垒3}

### 6. Messaging 矩阵
| 场景 | 说法 | 重点 |
|------|------|------|
| 电梯 30 秒 | | 痛点+方案 |
| 官网首页 | | 价值主张 |
| 投资人 | | 市场+壁垒 |
| 销售开场 | | 客户痛点 |
| 招聘 | | 使命+机会 |
```

## 异常处理
- CEO 定位摇摆 → 强制二选一："如果只能选一个客户群，你选谁？"
- 定位太宽 → 帮收窄，"什么都是"等于"什么都不是"
- 团队内部对定位有分歧 → 先对齐 CEO，再由 CEO 对齐团队

## 边界
- 不做品牌视觉设计（→ CCO）
- 不做产品功能定义（→ CPO）
- 不做技术差异化论证（→ CTO）
