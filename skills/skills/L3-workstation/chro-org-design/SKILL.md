# 📍 组织架构

## 触发条件
- CEO纠结组织问题
- 季度组织回顾
- 业务方向调整需要组织跟进

## 输入要求
1. 当前组织结构和团队规模（必须）
2. 业务方向和战略重点（必须）
3. 当前痛点（可选）

## 执行步骤
1. 梳理当前组织结构
2. 分析与业务战略的匹配度
3. 识别问题：冗余/缺失/汇报线混乱
4. 给出调整建议和过渡方案

## 输出模板

### 组织架构建议

**现状诊断**
- 当前结构：xxx
- 与战略匹配度：高/中/低
- 核心问题：xxx

**调整建议**
1. xxx（影响范围：xx人）
2. xxx

**过渡方案**
- 阶段1（第1-2周）：xxx
- 阶段2（第3-4周）：xxx

**风险提示**
- xxx

## 异常处理
- 涉及裁员 → 先给法律风险提示，操作方案和CLO一起出
- CEO犹豫不决 → 给出明确建议+两种方案对比
- 组织变动影响士气 → 建议配套文化建设措施

## 边界
- 不做：对具体个人的能力评价（→ CEO自行判断）
- 不做：战略方向决策（→ CSO/CEO Coach）
