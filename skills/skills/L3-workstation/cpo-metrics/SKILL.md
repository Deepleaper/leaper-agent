# 📍 产品度量

## 触发条件
- 功能上线后第 1/3/7 天
- CEO 问"这个功能效果怎么样"
- 数据异常时

## 输入要求
1. 功能名称和上线时间
2. CEO 提供的产品数据（DAU/留存/转化等）
3. 预期目标值
4. 可选：历史基线数据

## 执行步骤
1. **收集数据**：提醒 CEO 提供关键指标
2. **对比预期**：实际 vs 目标
3. **分析原因**：达标/未达标的原因假设
4. **建议行动**：下一步做什么
5. **持续追踪**：按节奏追踪后续变化

## 输出模板
```
## 产品度量：{功能名} 上线第 {N} 天

### 核心指标
| 指标 | 目标 | 实际 | 达标 | 趋势 |
|------|------|------|------|------|
| | | | ✅/❌ | ↑/↓/→ |

### 判断
{一句话结论：好/不好/需要观察}

### 原因分析
- {假设1}
- {假设2}

### 建议行动
1. {动作}
2. {动作}

### 下次追踪
- 时间：{日期}
- 关注：{指标}
```

## 异常处理
- 数据缺失 → 提醒 CEO 补数据，不编造
- 数据不达预期 → 分析原因+给调整建议，不说"再看看"
- 指标定义不清 → 先和 CEO 对齐"成功长什么样"

## 边界
- 不做数据采集/治理（→ CDO）
- 不做技术性能分析（→ CTO）
- 不做财务 ROI（→ CFO）
