# 📍 流程优化

## 触发条件
- CEO 抱怨某流程低效/慢/乱
- CEO 描述一个重复性工作想优化
- 团队扩张导致原有流程失效

## 输入要求
1. 流程现状描述（步骤、参与人、耗时）
2. 痛点在哪里
3. 可选：团队结构、工具现状

## 执行步骤
1. **流程还原**：把 CEO 口述画成流程步骤
2. **瓶颈定位**：找到最慢/最容易出错/最依赖人的环节
3. **根因分析**：是流程问题、人的问题、还是工具问题
4. **优化方案**：给 2-3 个方案（激进/稳健/渐进）
5. **实施建议**：落地步骤和预期效果

## 输出模板
```
## 流程优化：{流程名}

### 现状
{步骤1} → {步骤2} → ... → {步骤N}
总耗时：{时间} | 参与人：{人数}

### 瓶颈
- 🔴 {瓶颈环节}：原因是{原因}

### 优化方案
| 方案 | 描述 | 预期效果 | 实施难度 |
|------|------|---------|---------|
| A 激进 | | | ⭐⭐⭐ |
| B 稳健 | | | ⭐⭐ |
| C 渐进 | | | ⭐ |

### 建议
选方案 {X}，因为{理由}。

### 落地步骤
1. {步骤}（{时间}）
2. {步骤}
```

## 异常处理
- 流程涉及跨部门 → 出协调方案+沟通话术，不承诺"协调"本身
- 流程不完整 → 问 CEO 补充缺失环节
- CEO 想一步到位 → 建议分阶段，先解决最痛的点

## 边界
- 不做技术架构优化（→ CTO）
- 不做产品流程设计（→ CPO）
- 不做人事流程（→ CHRO）
