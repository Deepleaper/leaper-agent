# 📍 资源分配

## 触发条件
- CEO 纠结"人不够怎么分"
- 新项目启动需要调配资源
- 多项目并行优先级冲突

## 输入要求
1. 团队现状（人数、能力分布）
2. 项目清单和优先级
3. 各项目资源需求
4. 可选：预算约束、时间约束

## 执行步骤
1. **盘点现状**：当前人在做什么
2. **需求梳理**：各项目需要什么资源
3. **缺口分析**：供给 vs 需求的差距
4. **方案设计**：2-3 个分配方案
5. **代价说明**：每个方案的取舍
6. **CEO 决策**：呈现选项让 CEO 选

## 输出模板
```
## 资源分配建议

### 当前资源盘点
| 角色 | 人数 | 当前投入项目 | 可调配 |
|------|------|------------|--------|
| | | | |

### 项目资源需求
| 项目 | 优先级 | 需要 | 缺口 |
|------|--------|------|------|
| | P0/P1/P2 | | |

### 分配方案
| 方案 | 描述 | 代价 |
|------|------|------|
| A | | {砍什么} |
| B | | {砍什么} |

### 建议
选方案 {X}，因为{理由}。
```

## 异常处理
- 资源严重不足 → 明确说"资源不够同时做，必须砍项目"
- CEO 什么都不想砍 → 列出同时做的代价（延期、质量下降）
- 需要招人 → 转 CHRO 出 JD

## 边界
- 不做招聘决策（→ CHRO）
- 不做预算审批（→ CFO）
- 不做技术评估（→ CTO）
