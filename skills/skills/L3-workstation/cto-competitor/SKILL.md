# 📍 竞品技术分析

## 触发条件
- CEO 提到竞品
- tech-radar 发现竞品技术动态
- 竞品发布新产品/新功能

## 输入要求
1. 竞品名称和产品
2. 公开信息（GitHub/博客/招聘 JD/论文/API 文档）
3. 我方技术栈（做对比用）

## 执行步骤
1. 收集竞品公开技术信息：
   - GitHub 仓库（语言、框架、提交频率）
   - 技术博客（架构文章、技术选型分享）
   - 招聘 JD（从岗位要求推测技术栈）
   - 论文/专利（核心算法）
   - API 文档（接口设计思路）
2. 推测技术栈和架构
3. 分析技术差异化（他们有什么我们没有的）
4. 评估对我们的启示和威胁
5. 给出建议

## 输出模板

```markdown
# 竞品技术分析 | [竞品名]

## 技术栈推测
| 层面 | 推测技术 | 依据 |
|------|---------|------|
| 后端 | ... | [来源] |
| 前端 | ... | [来源] |
| AI/ML | ... | [来源] |
| 基础设施 | ... | [来源] |

## 技术差异化
- **他们有，我们没有**：[列表]
- **我们有，他们没有**：[列表]
- **双方都有但实现不同**：[对比]

## 威胁评估
- [具体威胁] — 严重程度：高/中/低

## 启示与建议
1. [可以借鉴的]
2. [需要防御的]
3. [我们的优势要保持的]

⚠️ 标注"推测"的部分基于公开信息推断，不保证准确。
```

## 异常处理
- 竞品信息不公开 → 基于招聘 JD 和博客推测，明确标注
- 竞品是开源项目 → 直接分析代码

## 边界
- 只看技术层面（→ 市场/产品分析找 CMO/CPO）
- 不做商业模式分析（→ CSO）
