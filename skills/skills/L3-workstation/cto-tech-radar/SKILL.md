# 📍 技术雷达

## 触发条件
- 定时：每日 08:00
- CEO 分享技术链接
- 行业重大技术发布

## 输入要求
1. 技术媒体源（HN/arXiv/GitHub Trending）
2. CEO 分享的链接
3. 公司技术栈档案（判断相关性）

## 执行步骤
1. 扫描信息源：HackerNews Top、arXiv AI 新论文、GitHub Trending
2. 过滤：只保留与公司业务/技术栈相关的信号
3. 分级：
   - 🔴 紧急：直接影响现有业务（竞品发布、依赖库漏洞）
   - 🟡 重要：中期可能影响技术决策
   - 🟢 关注：长期趋势，暂不需行动
4. 撰写日报
5. 重大突破时单独预警 CEO

## 输出模板

```markdown
# 技术日报 | YYYY-MM-DD

## 🔴 紧急
- [无 / 事件描述 + 影响分析 + 建议行动]

## 🟡 重要
1. **[标题]** — [一句话摘要]
   - 相关性：[与我们的关系]
   - 来源：[链接]

## 🟢 关注
- [标题]：[一句话]
- ...

## CTO 观点
[一段话总结今日趋势对我们的意义]
```

## 异常处理
- 搜索无结果 → "今日无重大技术动态"
- 信息矛盾 → 标注不确定性，列出多方观点
- 来源不可靠 → 标注"未经验证"

## 边界
- 不分析市场/商业动态（→ CMO/CSO）
- 只看技术层面，不看产品层面（→ CPO）
