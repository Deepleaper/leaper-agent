# 📍 客户提案（含 CRO 职能）

## 触发条件
- CEO 跟进大客户需要提案
- 招标/竞标场景
- CEO 问"帮我出个方案给客户"

## 输入要求
1. 客户名称和行业
2. 客户需求（明确的/推测的）
3. 产品能力匹配点
4. 定价策略/区间
5. 可选：竞品方案、客户预算

## 执行步骤
1. **客户调研**：搜索客户公司背景、行业痛点
2. **需求匹配**：把客户需求对应到产品能力
3. **价值论述**：不说"我们有什么"，说"你能得到什么"
4. **方案撰写**：按提案模板输出
5. **CEO 审核**：标注需要确认的报价数字
6. **竞标差异化**：如果知道竞品方案，突出差异点

## 输出模板
```
## 客户提案：{客户名}

### 1. 理解您的需求
{用客户的语言描述他们的痛点}

### 2. 方案概述
{一句话方案}

### 3. 价值匹配
| 您的需求 | 我们的方案 | 预期效果 |
|----------|-----------|---------|
| | | |

### 4. 实施方案
| 阶段 | 内容 | 周期 |
|------|------|------|
| 一期 | | |
| 二期 | | |

### 5. 投资与回报
- 方案报价：{⚠️ 需 CEO 确认}
- 预期 ROI：{估算}
- 付款方式：{建议}

### 6. 为什么选我们
1. {差异化1}
2. {差异化2}
3. {差异化3}

### 7. 下一步
- {具体的下一步行动}
```

## 异常处理
- 报价需要精确数字 → "报价数字需要你确认，我先出框架和价值论述"
- 不了解客户行业 → 先做行业调研再出方案
- 竞标时间紧 → 出精简版（3 页以内）

## 边界
- 不做合同法律条款（→ CLO）
- 不做报价的财务影响分析（→ CFO）
- 不做产品功能承诺（→ CPO 确认）
