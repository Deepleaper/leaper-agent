# 📍 竞品监控

## 触发条件
- 🕐 定时：每周三 10:00 扫描
- CEO 提到竞品名字
- 竞品发生融资、发布、并购等重大事件

## 输入要求
1. 竞品清单（至少 3-5 家）
2. 关注维度（定价、客户、渠道、融资、产品动态）
3. 可选：竞品历史数据

## 执行步骤
1. **搜索扫描**：搜索竞品最新公开信息
2. **变化检测**：对比上周，标注新变化
3. **影响判断**：对我方的影响（利好/利空/无影响）
4. **策略建议**：跟/不跟/观望，给理由
5. **同步相关工位**：技术层面同步 CTO，产品层面同步 CPO

## 输出模板
```
## 竞品周报 {日期}

### 本周重大动态
| 竞品 | 事件 | 影响 | 建议 |
|------|------|------|------|
| | | 🟢利好/🔴利空/⚪无影响 | 跟/不跟/观望 |

### 无重大变化的竞品
{列表}

### CMO 判断
{1-2 句总结}
```

## 异常处理
- 无重大动态 → "本周竞品无重大市场变化"，不硬凑
- 竞品融资 → 重点分析对我方影响，给 CEO 行动建议
- CEO 因竞品焦虑 → 用事实降温，列 3 个事实

## 边界
- 不做技术栈分析（→ CTO）
- 不做产品功能拆解（→ CPO）
- 不做竞品财务分析（→ CFO）
