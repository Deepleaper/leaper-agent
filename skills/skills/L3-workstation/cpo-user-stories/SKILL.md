# 📍 用户故事 / Use Case

## 触发条件
- 新需求需要细化场景
- CEO 说"用户会怎么用这个"
- PRD 撰写前的前置步骤
- 需要和 CTO 对齐技术方案时

## 输入要求
1. 需求卡片（来自 requirement 工位）
2. 目标用户画像
3. 可选：用户访谈记录、竞品用户流程

## 执行步骤
1. **角色识别**：这个功能涉及哪些用户角色
2. **核心故事**：每个角色的主线用户故事
3. **边缘场景**：异常流程、边界条件
4. **验收条件**：每个故事的 Given-When-Then
5. **优先级排序**：核心 vs 锦上添花
6. **CEO 确认**：确认覆盖度

## 输出模板
```
## 用户故事集：{功能名}

### 角色定义
| 角色 | 描述 | 使用频率 |
|------|------|---------|
| | | |

### 核心用户故事（Must-have）

#### US-1：{故事名}
- **作为**{角色}
- **我想**{做什么}
- **这样**{得到什么价值}

**验收条件：**
- Given {前提}
- When {操作}
- Then {结果}

#### US-2：{故事名}
...

### 边缘场景（Nice-to-have）

#### US-N：{故事名}
...

### Use Case 图（文字描述）
```
{角色} → {动作1} → {系统响应}
         → {动作2} → {系统响应}
         → {异常} → {系统处理}
```

### 优先级
| 故事 | 优先级 | 理由 |
|------|--------|------|
| US-1 | P0 | {理由} |
| US-2 | P1 | |
```

## 异常处理
- CEO 描述场景太笼统 → 追问"具体是谁在什么情况下遇到的？"
- 场景互相矛盾 → 列出矛盾点让 CEO 选
- 覆盖度不够 → 主动问"还有什么用户/场景我们漏掉了？"

## 边界
- 不做技术实现方案（→ CTO）
- 不做 UI 设计（→ 设计师）
- 不做市场验证（→ CMO）
