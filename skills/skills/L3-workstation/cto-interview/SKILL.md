# 📍 技术面试支持

## 触发条件
- CEO 要招技术人
- CHRO 转来技术岗需求

## 输入要求
1. JD 或岗位描述
2. 团队现有技术栈
3. 岗位级别（初级/中级/高级/架构师）
4. 可选：团队短板、期望候选人特质

## 执行步骤
1. 明确"这个人来了做什么"（如 JD 模糊先帮 CEO 明确）
2. 根据技术栈和级别设计面试题：
   - 基础知识题（筛选用）
   - 系统设计题（中高级）
   - 编码题（实际问题导向）
   - 行为面试题（团队协作、问题解决）
3. 为每道题设计评估标准（优秀/合格/不合格的表现）
4. 输出面试流程建议

## 输出模板

```markdown
# 技术面试方案 | [岗位名]

## 岗位核心要求
- [技能1]
- [技能2]
- [软素质]

## 面试流程（建议 3 轮）

### 第 1 轮：技术基础（30min）
| # | 题目 | 考察点 | 优秀表现 | 不合格表现 |
|---|------|--------|---------|-----------|
| 1 | ... | ... | ... | ... |

### 第 2 轮：系统设计（45min）
**题目：** [设计一个...]
**评估维度：**
- 需求澄清能力
- 架构合理性
- Trade-off 分析
- 扩展性考虑

### 第 3 轮：行为面试（30min）
1. "描述一次你解决过的最复杂的技术问题"
2. "你和同事技术方案有分歧时怎么处理？"

## 红旗信号 🚩
- [不该录用的信号]
```

## 异常处理
- 岗位要求模糊 → 先问 CEO"这个人来了第一个月做什么？"
- 技术栈冷门 → 搜索后设计，标注"我对这个技术了解有限"

## 边界
- 不做薪酬对标（→ CHRO）
- 不做最终录用决策（→ CEO）
