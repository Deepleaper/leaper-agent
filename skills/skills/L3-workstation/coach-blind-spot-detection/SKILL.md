# 📍 盲点探测

## 触发条件
- 认知日志中发现 CEO 持续回避某话题
- CEO 的决策模式出现明显偏差（过度乐观/悲观）
- 月度报告分析中发现盲点

## 输入要求
1. 近 30 天认知日志
2. CEO 决策模式档案
3. CXO 反馈信号（如有）

## 执行步骤
1. 分析认知日志，寻找：
   - **回避模式**：哪些话题 CEO 从不主动提？
   - **确认偏误**：CEO 是否只关注支持自己观点的信息？
   - **过度自信**：哪些判断缺乏数据支撑但 CEO 很确定？
   - **锚定效应**：CEO 是否被某个数字/事件锚定了？
2. 不直接指出盲点，而是通过提问引导：
   - "你最近很少提[某话题]，是有意放下还是没注意到？"
   - "你对[X]的判断很确定。有什么数据支撑吗？"
   - "除了[CEO 关注的]，还有什么可能影响结果？"
3. 记录 CEO 的反应（防御/接受/忽略）
4. 如果 CEO 防御 → 不追，下次再试

## 输出模板

```markdown
## YYYY-MM-DD 盲点观察

**盲点类型：** [回避/确认偏误/过度自信/锚定]
**表现：** [具体行为证据]
**引导尝试：** [用了什么问题]
**CEO 反应：** [防御/接受/忽略]
**后续策略：** [继续观察/下次换角度/已解决]
```

## 异常处理
- CEO 明确表示不想讨论 → 尊重，记录后等待时机
- 盲点涉及敏感话题（如联合创始人分歧）→ 非常温和地试探

## 边界
- 不强迫 CEO 面对盲点
- 不评判 CEO 的回避是对是错
- 盲点观察记录仅 Coach 内部使用，不发给 CEO
