# 📍 预算编制

## 触发条件
- 季度初
- CEO 要求做预算
- 战略方向变化需要重新分配资源

## 输入要求
1. 公司阶段（种子/天使/A/B 轮）
2. 行业和团队规模
3. CEO 战略目标（下季度重点）
4. 可选：历史支出数据、行业对标数据

## 执行步骤
1. 确认预算周期（季度/年度）和战略重点
2. 搜索行业对标数据（同阶段/同行业公司的支出结构）
3. 按大类拆解预算：
   - 人力成本（通常 60-70%）
   - 技术/基础设施（10-20%）
   - 市场/销售（10-20%）
   - 行政/办公（5-10%）
4. 根据战略重点调整分配比例
5. 设定预算红线和弹性空间

## 输出模板

```markdown
# 预算方案 | YYYY-Q[N]

## 预算基础
- 总预算：¥[X]/季度
- 公司阶段：[X]
- 团队规模：[X] 人
- 战略重点：[X]

## 预算分配

| 类别 | 金额 | 占比 | 行业中位数 | 备注 |
|------|------|------|-----------|------|
| 人力 | ¥[X] | X% | 65% | ... |
| 技术 | ¥[X] | X% | 15% | ... |
| 市场 | ¥[X] | X% | 12% | ... |
| 行政 | ¥[X] | X% | 8% | ... |

## 行业对标
[同阶段公司的典型支出结构参考]

## 弹性空间
- 可压缩项：[哪些可以少花]
- 不可压缩项：[哪些不能省]
- 预留应急：建议预留 [X]%

## ⚠️ 风险提示
- [预算紧张时的降级方案]
```

## 异常处理
- 行业数据不确定 → 给范围而非精确值
- CEO 战略不清晰 → 先帮 CEO 明确"下季度最重要的 3 件事"

## 边界
- 不做人员编制决策（→ CHRO）
- 不做技术选型（→ CTO）
- ⚠️ 以上为参考建议，重大决策请咨询专业顾问
