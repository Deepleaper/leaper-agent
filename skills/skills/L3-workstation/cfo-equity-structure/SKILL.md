# 📍 股权架构

## 触发条件
- CEO 考虑股权分配/调整
- 新一轮融资前需要模拟稀释
- 设立/调整期权池
- 合伙人进入/退出

## 输入要求
1. 当前股权结构（各股东持股比例）
2. 变动场景（融资/期权/新合伙人）
3. 可选：公司估值、期权池大小、融资金额

## 执行步骤
1. 梳理当前股权结构（Cap Table）
2. 根据场景模拟变动：
   - 融资稀释：新投资人占多少？各方怎么被稀释？
   - 期权池：从哪里出？稀释谁？
   - 股权转让：税务影响？
3. 计算变动后各方持股比例
4. 评估创始人控制权（>50%? >34%? >67%?）
5. 给出建议和风险提示

## 输出模板

```markdown
# 股权架构分析 | [场景描述]

## 当前 Cap Table
| 股东 | 持股比例 | 类型 |
|------|---------|------|
| 创始人 A | X% | 普通股 |
| 投资人 B | X% | 优先股 |
| 期权池 | X% | 预留 |

## 变动模拟：[场景]

### 变动前 → 变动后
| 股东 | 变动前 | 变动后 | 稀释 |
|------|--------|--------|------|
| 创始人 | X% | Y% | -Z% |
| ... | ... | ... | ... |

## 控制权分析
- 创始人控制权：[绝对控制 >67% / 相对控制 >50% / 一票否决 >34% / 警戒]
- 一致行动人安排：[有/无/建议]

## 关键数字
- 本轮稀释：[X]%
- 累计稀释：[X]%
- IPO 前预计创始人持股：[X]%（按 [N] 轮融资估算）

## 建议
1. [具体建议]
2. [风险提示]

## 需要 CLO 确认
- [涉及法律的事项：协议修改、工商变更等]
```

## 异常处理
- 股权结构复杂（多层架构）→ 先画清楚再分析
- 涉及代持/隐名 → 提示法律风险，转 CLO

## 边界
- 不做股权协议起草（→ CLO）
- 不做最终股权分配决策（→ CEO）
- 不做工商变更操作（→ CLO + 行政）
- ⚠️ 以上为参考建议，重大决策请咨询专业顾问
